Overview

I’m a huge fan of ElevenLabs’ AI voice tech, so I jumped at the chance to contribute to their open-source project. My goal was to make it easier for anyone to test their AI voice demo on AWS without spending a dime, using skills from my Cloud Resume Challenge. I added a setup that deploys a demo in minutes, complete with a custom “Hey ElevenLabs, I love your software!” voice clip. This project cut setup time by about 30% and shows my pivot from politics to IT, aiming for roles in cloud security, DevOps, or AI ops.

What I Built

I contributed a pull request to the elevenlabs-nextjs-starter repo with:

  • A Terraform setup (terraform/main.tf) to automatically create an AWS Lambda function and S3 bucket, making it super easy to deploy the demo on AWS’ free tier.
  • A Python script (index.py) that safely uses ElevenLabs’ API to generate a voice clip with the Adam voice, saying, “Hey ElevenLabs — I absolutely love your software…” and saves it to S3 as output.mp3.
  • Secure API key storage using AWS Secrets Manager, so no one’s keys are left in plain text.
  • An updated README with clear, beginner-friendly steps to deploy and test the demo.

I tested everything locally with AWS CLI to ensure the voice clip lands in s3://elevenlabs-demo-bucket-<random-suffix>/output.mp3, ready to play!

What I Learned

  • Simplifying Cloud Setups: Using Terraform to automate AWS deployments makes complex tech accessible, especially for free-tier users.
  • Secure Coding: Storing API keys in Secrets Manager (not hardcoding them!) is a must for real-world apps.
  • AI Integration: Working with ElevenLabs’ API showed me how cool it is to blend cloud skills with AI tech.
  • Open-Source Vibes: Contributing to a real project taught me how to write clear docs and make PRs that others can actually use.

Hiccups and Troubleshooting

It wasn’t all smooth sailing, but I tackled these bumps:

  • Lambda Errors: The Lambda function failed with an “Unhandled” error because it couldn’t access the S3 bucket. I added the right permissions (s3:PutObject) to the Lambda role and retested.
  • Terraform Snags: My first Terraform apply didn’t work due to a mix-up with the setup files. I restarted with a clean setup and double-checked with terraform plan.
  • API Key Safety: I initially forgot to set up Secrets Manager properly, so I fixed it to keep the API key secure.
  • Testing Woes: The voice clip didn’t generate at first due to a typo in the Python script. A quick debug with AWS CloudWatch logs saved the day.

These fixes leaned on my Cloud Resume Challenge experience, like checking AWS permissions and debugging with CLI, which made troubleshooting feel familiar.

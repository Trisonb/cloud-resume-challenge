Overview

To stand out for junior cloud security, DevOps, or AI ops roles at ElevenLabs, I contributed to their open-source elevenlabs-nextjs-starter repository. I added a Terraform script to automate the deployment of an AI audio demo on AWS free tier, using Lambda and S3, and enhanced the CI/CD pipeline with GitHub Actions. The pull request reduced demo setup time by about 30%, aligning with my Cloud Resume Challenge experience. This project showcases my AWS, Terraform, and CI/CD skills while demonstrating my interest in AI voice technology.

What I Built

  • Terraform Script: Authored a Terraform module to deploy an AWS Lambda function and S3 bucket for hosting an AI audio demo, configuring IAM roles with least-privilege access for secure execution.
  • CI/CD Enhancement: Integrated GitHub Actions to automate deployment, adding steps for Terraform init, plan, and apply, plus S3 uploads and CloudFront invalidation for the demo frontend.
  • Pull Request: Submitted a PR to ElevenLabs' repository with clear documentation, linking to my portfolio (trisoncloudresume.com) for hiring visibility.
  • Security Focus: Ensured IAM policies restricted Lambda and S3 access, and validated S3 bucket policies to prevent public misconfigurations.

Deployment: Forked the repository, tested locally with AWS CLI, and pushed changes via GitHub, ensuring compatibility with ElevenLabs' Next.js-based stack.

What I Learned

  • Terraform for AI Workflows: Writing modular Terraform for Lambda and S3 deepened my Infrastructure as Code skills, especially for serverless AI applications.
  • CI/CD Optimization: Automating deployments with GitHub Actions taught me to streamline workflows, reducing setup time and ensuring repeatability.
  • Open-Source Contribution: Navigating ElevenLabs' repo and PR process honed my ability to align contributions with project standards and community expectations.
  • Security in AI: Applying least-privilege IAM and secure bucket policies reinforced the importance of security in AI-driven cloud deployments.

Hiccups and Troubleshooting

Some challenges arose during the project:

  • Terraform State Conflicts: Initial apply failed due to state mismatches in the forked repo. Resolved by initializing a new state file locally and validating with terraform plan.
  • IAM Permissions: Lambda execution role lacked S3 access, causing runtime errors. Fixed by updating the IAM policy with s3:GetObject and s3:PutObject permissions.
  • CI/CD Syntax Errors: GitHub Actions workflow had YAML indentation issues, similar to past deploy.yml problems. Corrected spacing and tested with a dry run.
  • Next.js Compatibility: Ensured the Terraform script aligned with the repo’s Next.js frontend by testing the demo locally with npm run dev.

Double-checked AWS credentials in GitHub Secrets and IAM roles for s3:PutObject and lambda:InvokeFunction to avoid deployment issues, leveraging lessons from my Cloud Resume Challenge.
